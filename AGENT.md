## Operating promise

BITDOG separates planning from signing, dry-runs before wallet use, verifies every emitted hash on
chain, records no-tx failures explicitly, stops after the first failed write branch, and never calls
an open residual position a realized closeout.

The full identity and DOG thesis are in [SOUL.md](SOUL.md). Public documentation is not execution
authority. Only the operator and a current private approval scope can authorize a transaction.
